### 2.3 tasks 模块新增接口

```javascript
/**
 * 1. 获取每日挑战列表（供 challenges 模块调用）
 * @param {Array} excludeNames - 要排除的名称列表
 * @returns {Array} 每日挑战列表 [{taskId, subtask, taskName, taskDiff}]
 */
getDailyChallenges(excludeNames)

/**
 * 2. 切换子任务完成状态
 * @param {string} taskId - 父任务ID
 * @param {string} subId - 子任务ID
 * @param {boolean} done - 完成状态（可选，不传则切换）
 * @returns {Object|null} 更新后的子任务
 */
toggleSubtaskDone(taskId, subId, done)

/**
 * 3. 添加子任务
 * @param {string} parentId - 父任务ID
 * @param {Object} subtaskData - 子任务数据 {n, diff, daily, ...}
 * @returns {Object|null} 添加的子任务
 */
addSubtask(parentId, subtaskData)

/**
 * 4. 获取长期任务列表
 * @returns {Array} 长期任务列表
 */
getLongTasks()

/**
 * 5. 根据ID获取任务
 * @param {string} taskId - 任务ID
 * @returns {Object|null} 任务对象
 */
getTaskById(taskId)
```

## 三、导航修复记录（2026-03-22）

### 3.1 问题
当用户之前选择过世界后，系统会记住选择（存在 LocalStorage），下次打开直接跳转到任务视图。但导航栏里没有返回世界选择的按钮，导致用户被困住。

### 3.2 修复方案
1. 在导航栏添加"🌍 世界"按钮 - 点击可返回世界选择器
2. 添加返回逻辑 - 清除当前世界选择，显示世界选择器

### 3.3 修改文件
- `Next/LifeGame/index.html` - 导航栏添加"世界"按钮

## 六、事件总线使用说明

所有模块通过 `LifeGame.emit()` 和 `LifeGame.on()` 通信：

```javascript
// 发送事件
LifeGame.emit('view:tasks', { filter: 'today' });
LifeGame.emit('task:updated', { id: 'task-1' });
LifeGame.emit('world:selected', { world: 'surface' });

// 接收事件
LifeGame.on('view:tasks', function(data) {
  console.log('切换到任务视图', data.filter);
});
```

**常用事件清单：**
- `view:tasks` - 切换到任务视图
- `view:challenges` - 切换到挑战视图
- `view:timeline` - 切换到时间轴视图
- `view:profile` - 切换到角色视图
- `view:guide` - 切换到向导视图
- `world:selected` - 世界选择完成
- `task:updated` - 任务数据更新
- `storage:changed` - 存储数据变化
