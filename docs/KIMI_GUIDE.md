# Kimi 协作指南

_此文件指导Kimi如何与Echo协作_

---

## 你的工作流程

1. **读取需求**
   - 从 `待实现需求\` 文件夹读取需求文档
   - 需求由Echo根据与Kuan的对话整理保存

2. **编程实现**
   - 在VS Code中独立完成代码编写
   - 不需要知道其他文件夹的内容

3. **记录进度**
   - 完成后在 `log-progress.md` 追加记录
   - 格式见下方"记录格式"

---

## 记录格式（log-progress.md）

每次完成工作后，在文件末尾追加以下内容：

```markdown
## YYYY-MM-DD HH:MM

**已完成** ✅：
- [x] 功能1（简要描述实现内容）
- [x] 功能2（简要描述实现内容）

**进行中** 🚧：
- [ ] 功能3（当前进度%，预计完成时间）
- [ ] 功能4（当前进度%，遇到什么问题）

**待完成** ⏳：
- [ ] 功能5（计划要做的）
- [ ] 功能6（待需求确认）

**今日问题/备注**：
- 遇到的技术难点（可选）
- 需要确认的需求（可选）
- 明日计划（可选）
```

**示例**：
```markdown
## 2026-03-22 14:30

**已完成** ✅：
- [x] 任务系统子任务嵌套渲染
- [x] 修复时间轴hover效果

**进行中** 🚧：
- [ ] 过期颜色渐变（80%，预计今天完成）
- [ ] 骰子宝箱记录保存（50%，调试中）

**待完成** ⏳：
- [ ] 全勤骰子奖励统计
- [ ] 灵魂形态进化UI

**今日问题/备注**：
- 本地存储数据格式需要统一，已记录待重构
- 明日优先完成过期颜色渐变
```

## 文件说明

| 文件/文件夹 | 用途 | 你的操作 |
|:---|:---|:---|
| `README.md` | 开发文件夹说明 | 阅读 |
| **`log-progress.md`** | **写**你的工作记录 | **追加记录（唯一位置：桌面/开发/log-progress.md）** |
| `待实现需求\` | **读**Echo整理的需求 | **读取实现** |
| `LifeGame\` | 代码目录 | 自由操作 |
| `guide-*.md` | 技术指南 | 参考 |
| `project-*.md` | 项目总览 | 参考 |

---

## 协作边界

**你只管**：
- ✅ 读取 `待实现需求\` 的需求
- ✅ 编写代码到 `LifeGame\`
- ✅ 写进度到 `log-progress.md`

**你不用管**：
- ❌ 其他文件夹（游戏/健康/财务/视频）
- ❌ Echo与Kuan的对话内容
- ❌ 非技术类事务

---

## Echo会做什么

- 读取 `log-progress.md` 了解你的进度
- 向Kuan汇报开发状态
- 将Kuan的新需求整理到 `待实现需求\`

**你们通过文件间接协作，不需要直接对话。**
