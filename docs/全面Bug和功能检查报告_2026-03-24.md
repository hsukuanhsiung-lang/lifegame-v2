## 🔴 严重 Bug (需立即修复)

### 1. tasks/index.js - attachEvents 方法重复定义 ❌

**位置**: 第218-255行 和 第1010-1242行

**问题**: 两个同名方法，第二个完全覆盖第一个，导致第一个定义中的所有事件绑定逻辑永远不会执行

**影响**: 
- 返回按钮事件可能不工作
- 一些全局事件监听器未绑定

**修复方案**: 合并两个方法
```javascript
attachEvents: function() {
  var self = this;
  
  // 第一个 attachEvents 的逻辑
  var backBtn = document.getElementById('back-btn');
  if (backBtn) {
    backBtn.addEventListener('click', function() {
      LifeGame.emit('nav:back');
    });
  }
  
  // ... 其他逻辑
  
  // 第二个 attachEvents 的逻辑
  // ... 原有逻辑
}
```

---

### 2. tasks/index.js - XSS 安全风险 ❌

**位置**: 多处，如第752-761行、第792-809行

**问题**: 直接将用户输入（任务名称）插入 HTML，未进行转义
```javascript
return '<div class="task-name-grid">' + t.n + '</div>'  // t.n 可能包含 <script>
```

**影响**: 恶意脚本注入风险

**修复方案**: 
```javascript
// 添加转义函数
escapeHtml: function(text) {
  if (!text) return '';
  return String(text)
    .replace(/&/g, '&amp;')
    .replace(/</g, '&lt;')
    .replace(/>/g, '&gt;')
    .replace(/"/g, '&quot;')
    .replace(/'/g, '&#039;');
},

// 使用
'<div class="task-name-grid">' + this.escapeHtml(t.n) + '</div>'
```

### 3. challenges/index.js - `item.todayDate` 未定义 ❌

**位置**: 第 1939 行 `undoForbidViolation` 方法

**问题**: 
```javascript
item.history = item.history.filter(function(h) {
  return h.date !== item.todayDate;  // ❌ todayDate 从未定义
});
```

**影响**: 撤销违规功能可能无法正常工作

**修复方案**:
```javascript
var today = this.getTodayStr();
item.history = item.history.filter(function(h) {
  return h.date !== today;
});
```
