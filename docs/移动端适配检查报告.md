## 🔴 高优先级问题（必须修复）

### 1. 拖拽排序功能

**问题描述：**
- 使用 HTML5 Drag and Drop API (`draggable`, `dragstart`, `dragover`, `drop`)
- iOS Safari 完全不支持 HTML5 拖拽 API
- Android Chrome 支持有限，体验不佳

**影响文件：**
```
core/drag-sort.js
modules/tasks/index.js (第1352-1376行)
modules/challenges/index.js (第1349-1356行)
```

**移动端替代方案：**
```javascript
// 方案1：使用 touch 事件实现拖拽
// 方案2：使用长按菜单 + 上下移动按钮
// 方案3：引入 Sortable.js 等支持移动端的库

// 推荐：Sortable.js (https://github.com/SortableJS/Sortable)
// 它内部处理了 touch 事件和 HTML5 Drag API 的兼容
```

**修改建议：**
1. 引入 Sortable.js 替换当前 drag-sort.js
2. 或添加触摸事件支持，检测是否为触摸设备切换交互方式

---

### 2. 双击事件 (dblclick)

**问题描述：**
- 移动端没有"双击"手势，用户无法触发相关功能
- 当前有 6 处使用 dblclick

**影响位置：**

| 文件 | 行号 | 功能 | 建议替代方案 |
|-----|------|------|-------------|
| `nav-controller.js` | 346 | Day 徽章双击显示统计页 | 改为单击或长按菜单 |
| `tasks/index.js` | 1246 | 双击任务卡片编辑 | 改为长按菜单或详情页按钮 |
| `tasks/index.js` | 2507 | 双击子任务名称编辑 | 改为长按菜单 |
| `tasks/index.js` | 2576 | 双击"子任务"标签进入全貌 | 改为单击或菜单选项 |
| `challenges/index.js` | 116 | 绑定双击事件 | 检查是否需要 |
| `action/index.js` | 540 | 双击显示编辑按钮 | 改为长按菜单 |

**修改示例：**
```javascript
// 当前代码（仅桌面端有效）
element.addEventListener('dblclick', handler);

// 移动端兼容方案：长按检测
let pressTimer;
element.addEventListener('touchstart', (e) => {
  pressTimer = setTimeout(() => {
    handler(e); // 长按触发
  }, 500);
});
element.addEventListener('touchend', () => {
  clearTimeout(pressTimer);
});
```

## 🟡 中优先级问题（建议修复）

### 3. 鼠标悬停 Tooltip

**问题描述：**
- 今日任务卡片 hover 显示详情浮窗
- 移动端无鼠标悬停，无法查看任务详情

**影响文件：**
```
modules/tasks/index.js (第1289-1318行)
```

**当前代码：**
```javascript
card.addEventListener('mouseenter', function(e) {
  self.showTaskTooltip(task, card);
});
card.addEventListener('mouseleave', function() {
  self.hideTaskTooltip();
});
```

**移动端替代方案：**
1. **方案1（推荐）：** 点击卡片显示详情弹窗（替代 hover）
2. **方案2：** 添加"详情"按钮
3. **方案3：** 长按显示详情

**修改建议：**
```javascript
// 检测触摸设备
const isTouchDevice = 'ontouchstart' in window;

if (isTouchDevice) {
  // 触摸设备：点击显示 tooltip，点击其他地方隐藏
  card.addEventListener('click', (e) => {
    if (!e.target.closest('.task-checkbox')) {
      self.showTaskTooltip(task, card);
    }
  });
} else {
  // 桌面设备：保持 hover
  card.addEventListener('mouseenter', ...);
  card.addEventListener('mouseleave', ...);
}
```

## 🟢 低优先级问题（可选优化）

### 5. 原生弹窗样式 (alert/confirm/prompt)

**问题描述：**
- 共发现 40+ 处使用原生弹窗
- 移动端浏览器会拦截连续弹窗
- 样式不统一，无法自定义主题

**影响范围：**
- 所有模块都有使用

**建议：**
可选：创建自定义弹窗组件替代原生弹窗
```javascript
// 自定义确认弹窗
LifeGame.ui.confirm({
  title: '确认删除？',
  message: '此操作不可恢复',
  confirmText: '删除',
  cancelText: '取消',
  confirmClass: 'btn-danger'
}).then(result => {
  if (result) {
    // 执行删除
  }
});
```

**优先级：低**，因为原生弹窗在移动端功能正常，只是样式不美观。

### 6. 数据导出下载

**问题描述：**
- 使用 `URL.createObjectURL` + 自动点击下载链接
- 移动端浏览器可能阻止自动下载

**影响文件：**
```
modules/settings/index.js (第304-309行)
```

**当前代码：**
```javascript
var blob = new Blob([dataStr], { type: 'application/json' });
var url = URL.createObjectURL(blob);
var a = document.createElement('a');
a.href = url;
a.download = 'lifegame_backup_xxx.json';
a.click();
```

**移动端替代方案：**
1. 显示下载链接让用户手动点击
2. 复制到剪贴板让用户保存
3. 使用原生分享 API (Web Share API)

## 🔧 工具函数建议

添加移动端检测工具：

```javascript
// core/mobile-utils.js

LifeGame.mobile = {
  // 检测是否为触摸设备
  isTouchDevice: function() {
    return 'ontouchstart' in window || navigator.maxTouchPoints > 0;
  },
  
  // 检测是否为iOS
  isIOS: function() {
    return /iPad|iPhone|iPod/.test(navigator.userAgent);
  },
  
  // 添加长按事件监听
  addLongPressListener: function(element, handler, duration) {
    duration = duration || 500;
    let timer;
    
    element.addEventListener('touchstart', (e) => {
      timer = setTimeout(() => handler(e), duration);
    });
    
    element.addEventListener('touchend', () => clearTimeout(timer));
    element.addEventListener('touchmove', () => clearTimeout(timer));
  },
  
  // 阻止双击缩放
  preventDoubleTapZoom: function() {
    let lastTouchEnd = 0;
    document.addEventListener('touchend', (e) => {
      const now = Date.now();
      if (now - lastTouchEnd <= 300) {
        e.preventDefault();
      }
      lastTouchEnd = now;
    }, false);
  }
};
```
